## T1.2/T1.3 Freeze Requirements

- Generate actual byte-level goldens for prompt combinations and inline-agent continuations; source-snippet hashes alone are insufficient.
- Maintain the exhaustive runtime command/request/response/error fixtures for the 121/91 completeness gap; R3.1 / #351 closes the typing seam and R4.1–R4.4 complete handler cutover.
- Establish per-type legal and malformed bridge fixtures that expose current shallow validation, plus correlation, timeout, disconnect, and replay behavior; T2.1 owns receiver decoding and enforcement.
- Keep tool/sandbox failure and retry semantics explicit; never invent rollback for side effects that cannot be proven reversible.

## Current Evidence Limit

T1.2 replaces the source-snippet hashes with four raw UTF-8 golden files generated only from production functions. The executable cases cover composed memory/Skill/preset/project context, MCP and Shell descriptors, Chinese chat and English thinking prompts, direct and qualified XML names, inline continuation/nudge output, current truncation boundaries, the hidden continuation placeholder, and the current one-shot externalized-payload mismatch behavior. `npm run prompt:freeze` is read-only and reports a line diff; golden updates require the explicit `npm run prompt:freeze:update` command.

T2.1 turns the frozen bridge, tool-record, and sandbox fixtures into receiving-boundary codecs; adds browser-owned runtime sender/tab/frame/document context plus surface authorization; restricts content tab RPC and offscreen Port senders to the background entrypoint; and unifies the sandbox request size policy before privileged I/O.

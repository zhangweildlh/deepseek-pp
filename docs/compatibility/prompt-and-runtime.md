## T1.2/T1.3 Freeze Requirements

- Generate actual byte-level goldens for prompt combinations and inline-agent continuations; source-snippet hashes alone are insufficient.
- Establish an exhaustive runtime command/request/response/error fixture that documents the present 119/89 completeness gap; T3.1 eliminates the gap.
- Establish per-type legal and malformed bridge fixtures that expose current shallow validation, plus correlation, timeout, disconnect, and replay behavior; T2.1 owns receiver decoding and enforcement.
- Keep tool/sandbox failure and retry semantics explicit; never invent rollback for side effects that cannot be proven reversible.

## Current Evidence Limit

T1.2 replaces the source-snippet hashes with four raw UTF-8 golden files generated only from production functions. The executable cases cover composed memory/Skill/preset/project context, MCP and Shell descriptors, Chinese chat and English thinking prompts, direct and qualified XML names, inline continuation/nudge output, current truncation boundaries, the hidden continuation placeholder, and the current one-shot externalized-payload mismatch behavior. `npm run prompt:freeze` is read-only and reports a line diff; golden updates require the explicit `npm run prompt:freeze:update` command.

T1.3 adds executable request, response, error, and malformed fixtures for runtime commands, all 13 port bridge types, tool wire records, sandbox requests/results/errors, 17 notifications, three tab RPCs, and sandbox multi-hop envelopes. The runtime topology is derived through the TypeScript AST and paired with a 121-entry data-only contract registry rather than duplicated as a production router. Bridge sources, handshake types, and message names now have one authority; sandbox port/frame names, envelope parsing, result normalization, and boundary request normalization are likewise shared by their real producers and consumers. Malformed nested bridge/tool records, direct payload casts, the `32/2` command split, permissive shallow envelopes, and the sandbox/core size-policy split remain explicitly labeled current gaps owned by T2.1, T2.2, or T3.1.

These fixtures characterize existing behavior; they are not runtime codecs. Receiver hardening, exhaustive command decoding, and sandbox-validator unification remain follow-up work and must preserve the legal fixtures or make an explicit compatibility decision.

## DeepSeek++ 1.11.5

1.11.5 是一次聊天、MCP 与 Shell Host 稳定性更新，重点减少请求格式变化、重复刷新和长时间本机工具调用带来的中断。

### 主要变化

- **基础聊天不中断**：当某类 DeepSeek 请求无法由扩展安全增强时，会直接按原请求继续发送，避免普通聊天被扩展拦截或中止。
- **MCP 权限确认恢复**：在侧边栏连接需要网站权限的 MCP 服务时，会由当前用户操作直接发起浏览器权限确认，减少点击连接后无法正常授权的问题。
- **MCP 工具列表更稳定**：同一服务的测试和刷新会按顺序执行；刷新失败时保留最近一次成功发现的工具，同时明确显示连接错误。
- **Shell Host 保持响应**：长时间运行本机工具、初始化 Windows 命令环境或打开文件夹选择器时，工具列表和其他控制请求仍可及时响应，返回结果也会稳定输出。
- **权限变化**：Chrome、Edge 和 Firefox 均不新增浏览器权限。

### 验证

- `npm run ci:quality`
- `npm pack --dry-run --workspace packages/shell-host`
- `node scripts/web-search-smoke.mjs`

### Chrome Web Store 更新说明

中文：

- 遇到无法增强的请求时继续发送原请求，减少普通聊天被中断的情况。
- 修复部分 MCP 服务无法正常弹出网站权限确认的问题。
- MCP 刷新失败时保留最近可用的工具列表，并让重复测试和刷新更稳定。
- Shell Host 在长时间本机工具调用和 Windows 初始化期间仍能响应；Chrome 版本不新增浏览器权限。

English:

- Sends the original request when it cannot be safely enhanced, reducing interruptions to everyday chat.
- Fixes site-permission prompts not appearing correctly for some MCP service connections.
- Keeps the last available MCP tool list after a failed refresh and makes repeated tests and refreshes more reliable.
- Keeps the Shell Host responsive during long native tool calls and Windows startup work; the Chrome package adds no new browser permissions.

## DeepSeek++ 1.0.8

1.0.8 修复本地 Skill 导入、长内容写入配额、以及 Agent 并发三个用户反馈强烈的问题，重点让长时间任务更稳、错误提示更可操作。

### 主要变化

- **本地 Skill 导入兼容中文与 BOM**：导入带 UTF-8 BOM 的 `SKILL.md` 不再丢失 frontmatter，中文名/中文目录也能导入（自动生成稳定 slug 兜底，之后可在 Skills 界面改名）；Windows 反斜杠路径正确解析。
- **工具记录存储配额治理**：工具调用历史改用预算式写入，写满前自动裁剪最旧记录，不再每次工具调用都刷「QUOTA_BYTES」告警；单条记录的详情/输出快照收紧，历史上限从 200 降到 100。
- **本机写入大小对齐真实上限**：本机消息和 `local_file_write` 上限对齐 Chrome native messaging 的 ~1 MB 真实天花板，过大的写入会在插件侧和本机 host 两侧都被提前拦截，并明确提示「分块写入：先写前半部分，再用 `append=true` 写后续」；本机 host 响应接近上限时记录诊断日志。
- **Agent 并发保护**：Agent 运行期间，用户在对话框发送的新消息不再触发第二个重复 Agent；旧 Agent 面板在切换时同步移除，避免两个面板同时出现；新增「Agent 正在执行中」的状态提示。
- **回归覆盖**：新增 BOM 导入、中文 slug 兜底、历史预算裁剪、本机 900 KB 边界、Agent 并发守卫相关测试。

### 验证

- `npm run compile`
- `npm test`
- `npm run build:chrome`
- `npm run build:all`

### Chrome Web Store 更新说明

中文：

- 修复带 UTF-8 BOM 的本地 Skill（如 Windows 记事本保存的 SKILL.md）导入时丢失名称导致的「missing a valid name」报错；中文标题或中文目录的 Skill 现在可正常导入并自动生成可改名的标识。
- 工具调用历史写满存储时不再每次报配额错误，会自动清理最旧记录；长内容写入超过本机消息上限时会提示分块写入，而不是静默失败。
- Agent 长时间运行时，用户发送的后续消息不再触发重复 Agent 任务，避免同一对话里出现两个并行 Agent 面板。

English:

- Fixes local Skill imports dropping the `name:` field when `SKILL.md` starts with a UTF-8 BOM (e.g. saved by Windows Notepad), which previously raised "missing a valid name"; Skills with Chinese titles or directories now import with an auto-generated, renameable slug.
- Tool call history no longer logs a quota warning on every call when storage fills up — it proactively trims the oldest records; content writes that exceed the native messaging cap now prompt for chunked writes instead of failing silently.
- Long-running Agents no longer spawn a duplicate Agent task when the user sends a follow-up message, preventing two parallel Agent panels from appearing in the same conversation.

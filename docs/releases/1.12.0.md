## DeepSeek++ 1.12.0

1.12.0 引入本地 Skill 管理（local-skill management）：让 Agent 在聊天里直接激活、调用与重定位存放在本地的 Skill 文件夹，并把本地 Skill 的执行边界（cwd）硬钉死到其目录，避免越界读写本地文件。

### 主要变化

- **本地 Skill 激活（隐式打分）**：新增「自动激活」开关（首条消息 / 每条消息），对本地索引 Skill 按用户输入打分，过阈值且显著领先时自动激活；支持「适用场景 / 不适用场景」标签（兼容全角与半角冒号）。
- **本地 Skill 导入与重定位（T8）**：支持把本地文件夹登记为 Skill 源；原文件夹挪动或改名后，可经 UI 重新选择路径重定位，保留 source.id 与既有用户设置、激活引用不断裂。
- **D1 路径改写**：对进入 Agent 上下文的 Skill 文本里的相对路径引用做双基探测绝对化，并加「防逃逸」containment 校验，越界引用不改写。
- **方案 A cwd 硬强制**：激活本地 Skill 时，其命令型工具调用（shell_exec / shell_session_begin）在 Native Host 侧以 skillDir 为工作目录执行，落实 D4 Local Execution Boundary。
- **4 个新 runtime 命令**：RELOCATE_LOCAL_SKILL_SOURCE、GET_SKILL_AUTO_ACTIVATION_SETTINGS、SAVE_SKILL_AUTO_ACTIVATION_SETTINGS、UPDATE_LOCAL_SKILL_SOURCE；命令清单契约测试与 inventory 文档已同步。
- **权限变化**：Chrome、Edge 和 Firefox 均不新增浏览器权限。

### Chrome Web Store 更新说明

中文：

- 新增本地 Skill 管理：可在聊天中直接激活、调用并管理存放在本地的 Skill 文件夹。
- 本地 Skill 执行时自动以该 Skill 目录为工作目录，避免越界读写本地文件。
- 提供「自动激活」开关，可按对话首条或每条消息智能匹配本地 Skill。

English:

- Adds local Skill management: activate, invoke, and manage locally stored Skill folders directly from chat.
- Local Skill execution is pinned to the Skill's own directory as the working directory, preventing out-of-bounds local file access.
- Adds an auto-activation toggle that intelligently matches local Skills on the first or every conversation message.

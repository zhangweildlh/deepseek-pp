## DeepSeek++ 1.10.0

1.10.0 改进本地 Skill 的资源导入与按需读取体验，让大型 Skill 的文件状态、读取条件和失败修复更清楚，同时提升 MCP 代理工具的参数准确性。

### 主要变化

- **资源状态更清楚**：本地 Skill 预览会区分已内嵌文件和保留在原目录、可按需读取的支持文件，不再用“省略”造成文件丢失的误解。
- **读取能力校验**：导入前会确认 Shell MCP 是否真的能读取按需资源；不可用时提供更新 Native Host、调整执行模式或检查连接的明确提示。
- **混合目录仍可导入**：单个 Skill 因读取器不可用而受阻时，同目录中不依赖该能力的其他 Skill 仍可继续选择和导入。
- **MCP 参数更准确**：MCP 工具输入参数的可选值会在发现、缓存和提示链路中完整保留，改善代理 MCP 工具的调用准确性。
- **权限边界不变**：本次更新不新增浏览器权限；按需资源仍保留在用户选择的本地 Skill 目录中。

感谢 [@zhangweildlh](https://github.com/zhangweildlh) 反馈本地 Skill 资源说明问题并提供复现材料。

### Chrome Web Store 更新说明

中文：

- 本地 Skill 预览现在明确区分已内嵌文件和可按需读取的支持文件。
- 导入前会验证 Shell MCP 的读取能力，并在不可用时提供清晰的修复提示。
- 单个 Skill 的读取器不可用时，不再阻塞同目录中的其他 Skill。
- 修复 MCP 工具参数选项丢失导致的调用问题。
- 本次更新不新增浏览器权限。

English:

- Local Skill previews now distinguish bundled files from supporting resources available on demand.
- Shell MCP reader availability is verified before import, with actionable recovery guidance when unavailable.
- A blocked Skill no longer prevents other Skills in the same folder from being imported.
- Fixed missing MCP tool parameter options that could affect calls through MCP proxies.
- No new browser permissions are introduced.

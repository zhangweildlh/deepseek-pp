## DeepSeek++ 1.11.6

1.11.6 是一次文件生成稳定性更新，重点避免 OfficeCLI 等 Skill 生成本地文档时，异常超大内容拖垮页面或在刷新后再次出现。

### 主要变化

- **文件生成不再拖垮页面**：对异常超大的工具输出增加硬性保护，超过安全范围后停止继续处理，避免长串编码文件数据造成页面卡死。
- **刷新后不再重复加载残留内容**：历史恢复会清理未完成、跨片段或格式带空白的工具输出，避免刷新后再次渲染大段残留内容。
- **明确 Artifact 使用边界**：Artifact 用于 UTF-8 文本、代码和由文本组成的项目包；Office、PDF、图片、压缩包等本地二进制文件应保留在生成路径并直接告知用户，不应编码后放入聊天。
- **权限变化**：Chrome、Edge 和 Firefox 均不新增浏览器权限。

### Chrome Web Store 更新说明

中文：

- 修复 OfficeCLI 等 Skill 生成本地文件时，异常超大输出可能导致页面卡死的问题。
- 刷新页面时会清理未完成或被拆分的工具输出，避免残留的大段内容再次出现。
- Artifact 明确用于文本和代码；Office、PDF、图片、压缩包等本地文件直接保留生成路径。Chrome 版本不新增浏览器权限。

English:

- Fixes page freezes caused by unusually large output when OfficeCLI and similar Skills create local files.
- Cleans up unfinished or split tool output during history restoration so stale large content does not return after a reload.
- Clarifies that Artifacts are for text and code, while local Office, PDF, image, and archive files remain at their generated path. The Chrome package adds no new browser permissions.
